# TgTodo — функционал и технологии

Краткое описание проекта: Telegram Mini App для задач с баллами, личными и семейными списками.

---

## Назначение

Пользователь ведёт задачи в Telegram, получает баллы за выполнение и копит их на личном или групповом счёте. Семья (или другая группа) может иметь общие задачи и общий «кошелёк» баллов.

---

## Функционал (MVP — реализовано)

### Пользователь и вход
- Авторизация через Telegram Mini App (`initData`, проверка подписи на BFF).
- Регистрация/обновление профиля по `telegram_id`, часовой пояс.
- Локальная разработка без Telegram: заголовок `X-Dev-Telegram-Id`.

### Группы (семья, команда)
- Создание группы, шестизначный код приглашения.
- Вступление по коду, роли: Owner, Admin, Member.

### Задачи
- **Личная** или **общая** (в контексте группы).
- Категории (личные и групповые).
- Повторение: без повтора / ежедневно / еженедельно (учёт периода при выполнении).
- Настраиваемые баллы за выполнение.
- Видимость личной задачи (выбор создателя): только я / участники группы / я и исполнитель.
- Назначение исполнителя, режим «любой участник» / «каждый отдельно».
- Отметка выполнения; повторное выполнение в том же периоде не начисляет баллы повторно.

### Геймификация
- **Два счёта:** личный и групповой.
- Начисление баллов за complete (личная задача → личный счёт, общая → групповой).
- Баланс не уходит в минус.
- Журнал операций (ledger).

### Интерфейс (Mini App)
- Главная: задачи на сегодня, балансы, выбор группы.
- Создание задачи, управление группами, история баллов.

### Telegram Bot
- Команды MVP и inline-создание задач через `@бот` в любом чате — см. **[BOT.md](./BOT.md)**, код: `src/Bot/TgTodo.Bot`.

## Архитектура

Микросервисы на **.NET 8** с **Clean Architecture** (Domain → Application → Infrastructure → Api). Общие контракты и building blocks — в `src/Shared`.

```text
Mini App (Blazor WASM) → BFF → микросервисы
                              ↘
Tasks ──(событие TaskCompleted)──→ RabbitMQ ──→ Gamification
```

| Сервис | Ответственность |
|--------|-----------------|
| **Identity** | Пользователи Telegram |
| **Groups** | Группы, участники, invite |
| **Tasks** | Задачи, категории, ACL, complete, Outbox |
| **Gamification** | Счета, ledger, обработка событий |
| **BFF** | Auth, агрегация API, раздача Mini App |

У каждого сервиса — своя БД PostgreSQL. Межсервисная связь: HTTP (синхронно) и **RabbitMQ** + **MassTransit** (асинхронно, идемпотентная обработка).

## Ключевые бизнес-правила

1. Два кошелька: личный и групповой — начисление зависит от типа задачи.
2. Баланс ≥ 0; списание ниже нуля запрещено.
3. Личные задачи: видимость задаёт создатель.
4. Анти-фарм: одно начисление за задачу на период (`period_key`: день / неделя / once).
